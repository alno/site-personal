В этой заметке я расскажу о фасаде для систем логирования, <b>SLF4J</b>, опишу его особенности и использование.

<b>SLF4J (Simple Logging Facade for Java)</b> - библиотека для логгирования, ставящая своей целью предоставить максимально простой, но при этом мощный фасад для различных систем логгирования на Java. Чем же это лучше чем использование какой-либо распространенной библиотеки для логгирования?

Если сравнить использование различных систем логгирования, как <b>log4j</b> и обощенных фасадов, таких как, например, <b>slf4j</b>, то можно найти следующие отличия:

Использование конкретной реализации:
<ul>
	<li>Зависимость от этой конкретной реализации</li>
	<li>Необходимость искать адаптеры, при необходимости использования компонент, разработанных, для других реализаций логгирования</li>
	<li>Меньше компонент в зависимостях</li>
</ul>

Использование фасада:
<ul>
	<li>Возможность использования различных реализаций</li>
	<li>Большое число компонент (фасад + реализация)</li>
	<li>Возможность быстро менять реализацию, возможно даже не меняя не строчки кода</li>
</ul>

Дополнительно к этому, SLF4J предоставляет возможность интегрировать компоненты, зависимые от других систем логирования (<b>Log4J</b>, <b>JCL</b>), подставляя реализации, направляющие логи этих систем в SLF4J.

Таким образом, небольшой ценой использование фасада для логирования значительно увеличивает возможности для интегрирования компонента в другие системы, что особенно важно, если неизвестно, где компонент будет использован (например, если это какая-та библиотека).
